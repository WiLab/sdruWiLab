step

Receive signal and control data from USRP® board

Syntax

[Y LEN] = step(H)
[Y LEN OVER] = step(H)
Y = step(H,FC)
Y = step(H,LO)
Y = step(H,GAIN)
Y = step(H,DECIM)

Description

[Y LEN] = step(H) receives signal and control data from a USRP® board using Universal Hardware Driver (UHD) packets. Output signal, Y, is a column vector of double or 16-bit integer values and LEN is the actual data length.

[Y LEN OVER] = step(H) outputs a value indicating that overruns occurred. This port is enabled when you set the OverrunOutputPort property to true. A value of zero indicates no data loss occurred. A value equal to or greater than 1 indicates data loss. Although the value reported does not represent the actual number of packets dropped, the higher this value, the farther you are from achieving real-time performance.

Y = step(H,FC) configures the USRP® board to receive at the center frequency, FC, when you set the CenterFrequencySource property to Input port. Center frequency, FC, must be a double scalar.

Y = step(H,LO) configures the USRP® board to apply a local oscillator offset, LO, when you set the LocalOscillatorOffsetSource property to Input port. The offset, LO, must be a double precision scalar.

Y = step(H,GAIN) configures the USRP® board to receive using a gain specified by the input, GAIN, when you set the GainSource property to Input port. Input, GAIN, must be a double scalar.

Y = step(H,DECIM) configures the USRP® board to receive using a decimation factor, DECIM, when you set the DecimationFactorSource property to Input port. Input, DECIM, must be an integer scalar.

    Note   The object performs an initialization the first time the step method is executed. This initialization locks nontunable properties and input specifications, such as dimensions, complexity, and data type of the input data. If you change a nontunable property or an input specification, the System object issues an error. To change nontunable properties or inputs, you must first call the release method to unlock the object.
