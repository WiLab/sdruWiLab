"Target for Use with USRP® Radio" Renamed to "Communications System Toolbox Support Package for USRP® Radio"

As of release R2013a, "Target for Use with USRP® Radio" is known as "Communications System Toolbox™ Support Package for USRP® Radio".

New Getting Started Section for Getting Your Radio Up and Running

The example "Getting Started with the SDRu" has been replaced with new documentation detailing the steps you must follow to connect your host computer with your USRP® hardware. See the Getting Started section on the main documenation page for Communications System Toolbox Support Package for USRP® Radio.

System Objects Support Offline Mode

You can program with and generate code for the Receiver and Transmitter System objects without a connection to a USRP® radio.

  • You can set the IPAddress property of System objects to any legal IP address value without having a connection to a USRP® radio at that IP address connected to your host computer.

  • You cannot use tab completion on the IPAddress property to get a list of connected USRP® radios. You can find all connected USRP® radios with the findsdru function.

  • You no longer have to set the IPAddress property as a value-only property for code generation.

  • You can generate code from these System objects even if you do not have a USRP® radio with the specified IP address connected to your host computer.

Updated Radio Discovery Function Also Returns Radio Status

Function findsdru now returns a structure containing the radio IP address and its current status.

Status messageMeaning
SuccessDevice is available.
Not compatibleDevice with incompatible firmware. To communicate with this device, you must update the firmware to the version returned by the function getSDRuDriverVersion(). Refer to Troubleshooting for instructions.
Not respondingDevice not responding. It could mean one of the following:

  • The device may not be attached to the computer.

  • There may not be a device with the specified IP address.

  • The subnet address of the host computer may not match the subnet address of the device (the subnet address is the third field of the IP address).

Busy Device is in use. The device is already owned by a block, block dialog, or System objectTM.
Unknown errorUnknown problem.

Added Functions for Getting Radio Information and Setting IP Addresses

Two new functions have been added:

  • probesdru: Provides detailed information about the USRP® radio connected to the host computer.

  • setsdruip: Sets the IP address of the USRP® device to a new IP address.

QPSK Receiver and Transmitter Examples for MATLAB

Examples are now available for developing a QPSK (Quadrature Phase Shift Keying) Receiver and QPSK Transmitter using Communications System Toolbox Support Package for USRP® Radio.

  • QPSK Receiver with USRP® Hardware: To open this example, enter showdemo('sdruQPSKReceiver') at the MATLAB® command line.

  • QPSK Transmitter with USRP® Hardware: To open this example, enter showdemo('sdruQPSKTransmitter') at the MATLAB command line.

Known Issues

Limitation When Using XCVR2450 Daughterboard

MATLAB will abort if you disconnect a USRP® device with an XCVR2450 daughterboard under one of the following conditions:

  • You are simulating a model that contains a Support Package for USRP® Radio Simulink block and the block dialog is open, or

  • You are using a Support Package for USRP® Radio System objectTM that is communicating with the USRP® hardware.

Multiple-Input Multiple-Output (MIMO) Support

Use Support Package for USRP® Radio in single-input, single output (SISO) applications. This software does not support the transmission of multiple, synchronized data streams to and from multiple USRP® transmitters and receivers.

Antenna Configuration

Antenna configurations for daughterboards remain in their default settings and cannot be changed. For more information, see Selecting an RF Daughterboard on the Ettus ResearchTM web site for details.
