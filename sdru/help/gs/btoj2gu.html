Possible Reasons and Solutions

The problem may be caused by one of the following:

  • An Ethernet connectivity problem

    If you are having an Ethernet connectivity problem, see Check Ethernet Configuration.

  • You have multiple versions of software on the host machine.

    Sometimes, in this case, you can still run the examples and get a proper IP address when you issue the call from <SDRuInstallRoot>/bin/<ARCH>. This is because the host is calling a different version of the driver than when you issue the function from <SDRuInstallRoot>.

    Examine the PATH environment variable to see if any system installs of software precede the Support Package for USRP® Radio install of software.

    • Linux: Examine LD_LIBRARY_PATH

    • Mac: Examine DYLD_LIBRARY_PATH

  • The radio is at an IP address that doesn't match your NIC. For example, your NIC may have an IP address of 192.168.10.1, while the USRP® radio may have an address of 192.168.20.2. Notice the difference in the third octet.

    Check that the subnet value of the host and radio are the same. See Check Subnet Values on Host and Radio.

  • Your anti-virus software may be blocking broadcast Ethernet packets.

    The UHDTM software uses broadcast packets to find radios. Disable your anti-virus software program or selectively stop the blocking of broadcast packets.
