step

Transmit signal and control data to USRP® board

Syntax

step(H,X)
UNDER = step(H,X)
step(H,X,FC)
step(H,X,LO)
step(H,X,GAIN)
step(H,X,INTERP)

Description

step(H,X) transmits signal and control data to a USRP® board using User Datagram Protocol (UDP) packets. The input signal, X, is a column vector of double values.

UNDER = step(H,X) outputs a value indicating that underruns occurred. This port is enabled when you set the UnderrunOutputPort property to true. A value of zero indicates no data loss occurred. Any value equal to or greater than 1 indicates data loss. Although the value reported does not represent the actual number of packets dropped, the higher this value, the farther you are from achieving real-time performance.

step(H,X,FC) configures the USRP® board to transmit at the center frequency, FC, when you set the CenterFrequencySource property to Input port. Center frequency, FC, must be a double precision scalar.

step(H,X,LO) configures the USRP® board to apply a local oscillator offset, LO, when you set the LocalOscillatorOffsetSource property to Input port. The offset, LO, must be a double precision scalar.

step(H,X,GAIN) configures the USRP® board to transmit using a gain specified by the input, GAIN, when you set the GainSource property to Input port. The Input, GAIN, must be a double precision scalar.

step(H,X,INTERP) configures the USRP® board to transmit using an interpolation factor, INTERP, when you set the InterpolationfactorSource property to Input port. Input, INTERP, must be an integer scalar.

You can combine optional input arguments when you set their enabling properties. Optional inputs must be listed in the same order as the order of the enabling properties. For example, step(H,X,FC,LO,GAIN,INTERP).

    Note   The object performs an initialization the first time the step method is executed. This initialization locks nontunable properties and input specifications, such as dimensions, complexity, and data type of the input data. If you change a nontunable property or an input specification, the System object issues an error. To change nontunable properties or inputs, you must first call the release method to unlock the object.
