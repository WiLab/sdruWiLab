comm.SDRuReceiver class

Package: comm

Receive data from USRP® device

Description

The comm.SdruReceiver System objectTM receives data from a Universal Software Radio Peripheral USRP®) hardware device, allowing simulation and development for various software-defined radio applications. The SDRuReceiver System objectTM enables communication with a USRP® board on the same Ethernet subnetwork. You can write a MATLAB® application that uses the System objectTM or you can generate code for the System objectTM without being connected to a USRP® radio.

This object receives signal and control data from a USRP® board using the Universal Hardware Driver (UHDTM) from Ettus ResearchTM. The SDRuReceiver System objectTM is a signal source that receives data from a USRP® board and outputs a column vector signal of fixed length. The first call to this object may contain transient values which result in packets containing undefined data.

Construction

H = comm.SDRuReceiver creates an SDRu Receiver System objectTM, H. This object supports communication between MATLAB and a USRP® board on the same Ethernet subnetwork, allowing simulation and development of various software-defined radio applications.

H = comm.SDRuReceiver(Name,Value) creates an SDRu Receiver object, H, with the specified property name set to the specified value. You can specify additional name-value pair arguments in any order as (Name1,Value1,...,NameN,ValueN).

H = comm.SDRuReceiver(ADDRESS,Name,Value) creates an SDRu Receiver object, H. This object has the IPAddress property set to ADDRESS and the other specified properties set to the specified values.

Properties

IPAddress

IP address of the USRP® device

Specify the logical network location of the USRP® device as a string. The default value is '192.168.10.2'. You can find all connected USRP® radios with the findsdru function.

CenterFrequencySource

Source of the center frequency

Specify the source of the center frequency as one of Property | Input port. The default is Property. Set CenterFrequencySource to Input port to specify the center frequency value using an input to the step method. Set CenterFrequencySource to Property to specify the center frequency value using the CenterFrequency property.

CenterFrequency

Center frequency in Hertz

Specify the desired center frequency as a double-precision, nonnegative, finite scalar. The default is 2.45 Gigahertz. The valid range of values for this property depends on the RF daughter card of the USRP® device. This property applies when you set the CenterFrequencySource to Property. This property is tunable.

ActualCenterFrequency

Actual center frequency in Hertz

Report the actual center frequency of the daughterboard. Desired and actual center frequency can be slightly different due to quantization. This property is read-only.

LocalOscillatorOffsetSource

Source of local oscillator offset

Specify the source of the local oscillator offset as one of Property | Input port. The default is Property. Set the LocalOscillatorOffsetSource property to Input port to specify the local oscillator offset value using an input to the step method. Set the LocalOscillatorOffsetSource property to Property to specify the local oscillator offset value using the LocalOscillatorOffset property.

LocalOscillatorOffset

Local oscillator offset in Hertz

Specify the desired local oscillator offset frequency as a double precision scalar. The default is 0 Hz. The valid range of this property depends on the RF daughterboard of the USRP® device. This property applies when you set the LocalOscillatorOffsetSource to Property. This property is tunable.

ActualLocalOscillatorOffset

Actual local oscillator offset in Hertz

Report the actual local oscillator offset of the daughterboard. Desired and actual offset can be slightly different due to quantization. This property is read-only.

GainSource

Source of the gain

Specify the source of the gain as one of Property | Input port. The default is Property. Set GainSource to Input port to specify the gain value using an input to the step method. Set GainSource to Property to specify the gain value using the Gain property.

Gain

Specify the overall gain of the USRP® hardware receiver data path

Specify the overall gain, both analog and digital, of the USRP® hardware receiver data path as a double scalar using decibels. The default is 8. The valid range of this property depends on the RF daughterboard of the USRP® device. This property applies when you set the GainSource to Property. This property is tunable.

ActualGain

Actual gain in decibels.

Report the actual gain of the daughterboard. Desired and actual gain can be slightly different due to quantization. This property is read-only.

DecimationFactorSource

Source of the decimation factor

Specify the source of the decimation factor as one of Property | Input port. The default is Property. Set DecimationFactorSource to Input port to specify the decimation factor value using an input to the step method. Set DecimationFactorSource to Property to specify the decimation factor value using the DecimationFactor property.

DecimationFactor

Decimation factor

Specify the decimation factor for the SDRu receiver as an integer scalar. The default is 512. Select an integer value within the following ranges:

  • 4 to 128

  • 128 to 256 (values in this range must be even)

  • 256 to 512 (values in this range must be evenly divisible by 4)

This property applies when you set the DecimationFactorSource to Property. This property is tunable.

ActualDecimationFactor

Actual decimation factor

Report the actual decimation factor value of the daughterboard. Desired and actual decimation factors can be slightly different due to quantization. This property is read-only.

OverrunOutputPort

Enable an output port that indicates dropped packets

Set this property to true so that the step method outputs a value to indicate if one or more packets is dropped during host reception of USRP® hardware data.

  • Zero indicates no data loss

  • A positive number indicates that overruns occurred

The default value for OverrunOutputPort is false, which means that the port is not enabled and no information about dropped packets is displayed.

This port is a useful diagnostic tool to determine real time operation of the . If your model is not running in real time, you can increase the decimation factor to approach or achieve real-time performance.

SampleRate

Specify the sample rate of the input signal, in Hertz, as a double-precision, positive, scalar value. The default is 1. Sample rate is the rate between successive samples of MATLAB data, not successive frames of data.

OutputDataType

Data type of output

Specify the data type of the output signal as double, single, or int16. The default is int16, with a range of [-32768, 32767]. This block supports the following complex output data types:

  • Double-precision floating point

  • Single-precision floating point

  • 16-bit signed integers

When you set OutputDataType to either Double-precision floating point or Single-precision floating point, the range is [-1, 1].

FrameLength

Frame length

Specify the frame length of the output signal that the object generates as a positive, scalar integer. The default value is 362. This value optimally utilizes the underlying Ethernet packets, which have a size of 1500 8-bit bytes.

EnableBurstMode

Turn burst mode on or off

When set to true, this property produces a set of contiguous frames without an overrun or underrun to the radio. This setting can help simulate models that cannot run in real time. When enabled, specify the desired amount of contiguous data using the NumFramesInBurst property. The default value is false.

NumFramesInBurst

Number of frames in contiguous burst

This property is valid when EnableBurstMode property is set to true. The default number of frames in a burst is 100.

Examples

Configure a USRP® board with an IP address of 192.168.10.2 to receive at 2.5 GHz with a decimation factor of 256. Save the data using a signal logger System objectTM.

    hSDRu = comm.SDRuReceiver('192.168.10.2', ...
              'CenterFrequency', 2.5e9, ...
              'DecimationFactor', 256);
    hLogger = dsp.SignalLogger;
    for counter = 1:20
      data = step(hSDRu);
      step(hLogger, data);
    end

For additional demonstrations that use the SDRu Receiver System objectTM, see the following examples:

  • FM Monophonic Receiver with USRP® Hardware: To open this example file, enter edit sdruFMMono at the MATLAB command line.

  • FM Stereo Receiver with USRP® Hardware: To open this example file, enter edit sdruFMStereoReceiver at the MATLAB command line.

  • FRS/GMRS Walkie-Talkie Receiver with USRP® Hardware: To open this example file, enter edit sdruFRSGMRSReceiver at the MATLAB command line.

Algorithms

This object implements the algorithm, inputs, and outputs described on the SDRu Receiver block reference page. The object properties correspond to the block parameters.
