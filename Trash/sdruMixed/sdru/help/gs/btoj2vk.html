USRP® radio is busy

Problem

MATLAB returns the following warning message from a call to function findsdru:

Warning: USRP radio at address '192.168.10.2' is busy. The radio is already owned by a block, block dialog, or System object. Stop the simulation, close the open dialog, or call the release method of the System object to allow a new owner to be assigned.

The USRP® radio is in use by another MATLAB or Simulink entity. USRP™ radios can become busy if any of the following conditions occur:

  • A Simulink simulation is in progress

  • A Support Package for USRP® Radio Receiver or Transmitter block mask is open

  • A locked Support Package for USRP® Radio Receiver or Transmitter System objectTM is present

Back to Top

Possible Solution

You can release the USRP® radio by stopping the simulation, closing the block mask, or calling the release method of the System objectTM.
