SDRu Receiver

Receive data from USRP® device

Library

Communications System Toolbox™ Support Package for USRP® Hardware

Description

The SDRu Receiver block supports communication between Simulink® and a Universal Software Radio Peripheral (USRP®) device, allowing simulation and development for various software-defined radio applications. The SDRu Receiver block enables communication with a USRP® board on the same Ethernet subnetwork.

This block receives signal and control data from a USRP® board using the Universal Hardware Driver (UHDTM) from Ettus ResearchTM. The SDRu Receiver block is a Simulink source that receives data from a USRP® board and outputs a column vector signal of fixed length. The first call to this block may contain transient values which result in packets containing undefined data.

The following block diagram illustrates how Simulink, SDRu blocks, and USRP® hardware interface.

When this block is called, it is possible that the host may not have received any data from the USRP® hardware. The data length port, Data Len, indicates when valid data is present. When the data length port contains a zero value, there is no data. You can use the data length with an enabled subsystem to qualify the execution of part of the model.

If your computer is not connected to any USRP® hardware, you can still use this block to develop a model that propagates sample time and data type information. To propagate this information, select Edit >> Update diagram; alternatively, you can press Ctrl + D.

For information about the USRP® hardware products with which the blocks interface, see the official Ettus ResearchTM Web site.

Dialog Box

USRP IP address:

This parameter contains a string that specifies the network location of the USRP® board. The default IP address is 192.168.10.2.

The drop-down list displays IP addresses for USRP® devices attached to the host computer. Alternatively, you can specify a known dotted quad IP address by entering it directly into this field.

Center frequency (Hz):

Specifies the center frequency of the input signal for the USRP® hardware front-end. You can specify this parameter using the block dialog mask or a block input port. You can specify this parameter using the block dialog mask or a block input port.

LO offset (Hz)

Specifies the offset frequency for the local oscillator. The valid range of this parameter depends on the RF daughterboard of the USRP® device.

The local oscillator offset does not affect the received center frequency. It does affect the intermediate center frequency in the USRP® hardware, as shown in the following illustration.

  • fRF represents the received RF frequency

  • fCenter represents the center frequency specified by the block

  • fLO offset is the local oscillator offset frequency

Use the LO offset to move the center frequency away from interference or harmonics generated by the USRP® hardware.

Gain (dB):

Specifies the overall gain, both analog and digital, of the USRP® hardware receiver data path.

Decimation:

Specifies the decimation factor for the SDRu Receiver block. The USRP® hardware uses this information to downconvert the Intermediate Frequency (IF) signal to complex baseband. Select an integer value within the following ranges:

  • 4 to 128

  • 128 to 256 (values in this range must be even)

  • 256 to 512 (values in this range must be evenly divisible by 4)

Enable Overrun output port

Select this check box to display the overrun port. This port indicates if one or more packets get dropped when the USRP® hardware transmits data to the host.

  • Zero indicates no packets dropped

  • A positive number indicates the number of packets dropped during simulation

This port can be used as a diagnostic tool to determine real time operation of the block. If your model is not running in real time, you can increase the decimation factor on the block mask to approach or achieve real-time performance.

Sample time:

Specifies Simulink sample time.

For the Simulink sample time to correspond to real time, use the following formula:

Sample time = M·1e-8

where M represents the value of the Decimation parameter.

If the decimation factor changes during simulation, then the hardware data rate also changes, but the Simulink sample time does not change.

The default setting for this parameter is 1.

Output data type

Select the data type of the output signal. The default is int16, with a range of [-32768, 32767]. This block supports the following complex output data types:

  • Double-precision floating point

  • Single-precision floating point

  • 16-bit signed integers

When you set Output data type to either Double-precision floating point or Single-precision floating point, the range is [-1, 1].

Frame length

Specify the frame length of the signal that the block outputs. The default setting for this parameter is 362. This value optimally utilizes the underlying Ethernet packets, which have a size of 1500 bytes.

Enable burst mode

Ensure a set of frames without overrun or underrun

When checked, this option produces a set of contiguous frames without an overrun or underrun to the radio. This setting can help simulate models that cannot run in real time. When enabled, specify the desired amount of contiguous data using the NumFramesInBurst option. The default value is not checked.

Number of frames in burst

Number of frames in contiguous burst

This option is valid when EnableBurstMode is checked. The default number of frames in a burst is 100.
