"Target for Use with USRP® Hardware" Renamed to "Target for Use with USRP® Radio"

As of release R2012b, "Target for Use with USRP® Hardware" is known as "Target for Use with USRP® Radio".

"Target for Use with USRP® Radio" Support Package Installer

You can now download the Target for Use with USRP® Radio software using the Support Package Installer function.

  1. From a MATLAB prompt, enter:

    >>targetinstaller

  2. Follow the prompts to download and install Target for Use with USRP®.

Perform 802.11 Beacon Reception with a Captured Signal Example

IEEE 802.11 WLAN - Beacon Frame Receiver with USRP® Hardware: To open this example, enter showdemo(‘sdruwlan80211BeaconRx') at the MATLAB command line. Simulink support only.

Known Issues

Limitation When Using XCVR2450 Daughterboard

MATLAB will abort if you disconnect a USRP® device with an XCVR2450 daughterboard under one of the following conditions:

  • You are simulating a model that contains a Target for Use with USRP® Radio Simulink block and the block dialog is open, or

  • You are using a Target for Use with USRP® Radio System objectTM that is communicating with the USRP® hardware.

Full-Duplex Support

Target for Use with USRP® Radio only supports half-duplex applications. MATLAB and Simulink applications must be receive-only or transmit-only. Full-duplex is not supported.

Multiple-Input Multiple-Output (MIMO) Support

Use Target for Use with USRP® Radio in single-input, single output (SISO) applications. This software does not support the transmission of multiple, synchronized data streams to and from multiple USRP® transmitters and receivers.

Antenna Configuration

Antenna configurations for daughterboards remain in their default settings and cannot be changed. For more information, see Selecting an RF Daughterboard on the Ettus ResearchTM web site for details.
