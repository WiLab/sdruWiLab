QPSK Examples

QPSK Transmitter Using USRP® Hardware: To open this example, enter sdruqpsktx at the MATLAB command line.

QPSK Receiver, with Synchronization, Using USRP® Hardware: To open this example, enter sdruqpskrx at the MATLAB command line.

Frequency Calibration Models for Use with QPSK Examples

  • Frequency Calibration Transmitter Using USRP® Hardware: To open this model, enter sdrufreqcalibtx at the MATLAB command line.

  • Frequency Calibration Receiver Using USRP® Hardware: To open this model, enter sdrufreqcalibrx at the MATLAB command line.

You can also open the models from within the QPSK examples.

Known Issues

Limitation When Using XCVR2450 Daughterboard

MATLAB will abort if you disconnect a USRP® device with an XCVR2450 daughterboard under one of the following conditions:

  • You are simulating a model that contains a Communications with USRP® Simulink block and the block dialog is open, or

  • You are using a Communications with USRP® System objectTM that is communicating with the USRP® hardware.

Full-Duplex Support

Communications with USRP® only supports half-duplex applications. MATLAB and Simulink applications must be receive-only or transmit-only. Full-duplex is not supported.

Multiple-Input Multiple-Output (MIMO) Support

Use Communications with USRP® software in single-input, single output (SISO) applications. This software does not support the transmission of multiple, synchronized data streams to and from multiple USRP® transmitters and receivers.

Antenna Configuration

Antenna configurations for daughterboards remain in their default settings and cannot be changed. For more information, see Selecting an RF Daughterboard on the Ettus ResearchTM web site for details.
