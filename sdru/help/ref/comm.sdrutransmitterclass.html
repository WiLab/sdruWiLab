comm.SDRuTransmitter class

Package: comm

Send data to USRP® device

Description

The SDRu Transmitter System objectTM sends data to a Universal Software Radio Peripheral (USRP®) hardware device, allowing simulation and development for various software-defined radio applications. The SDRu TransmitterSystem objectTM enables communication with a USRP® board on the same Ethernet subnetwork. You can write a MATLAB® application that uses the System objectTM or you can generate code for the System objectTM without being connected to a USRP® radio.

This object accepts a column vector input signal from MATLAB and transmits signal and control data to a USRP® board using the Universal Hardware Driver (UHDTM) from Ettus ResearchTM. The SDRu Transmitter System objectTM is a sink that sends the data it receives to a USRP® board. The first call to this object may contain transient values which result in packets containing undefined data.

Construction

H = comm.SDRuTransmitter creates an SDRu Transmitter System objectTM, H. This object supports communication between MATLAB and a USRP® board on the same Ethernet subnetwork, allowing simulation and development of various software-defined radio applications.

H = comm.SDRuTransmitter(Name,Value) creates an SDRu Transmitter object, H, with the specified property Name set to the specified Value. You can specify additional name-value pair arguments in any order as (Name1,Value1,...,NameN,ValueN).

H = comm.SDRuTransmitter(ADDRESS,Name,Value) creates an SDRu Transmitter object, H. This object has the IPAddress property set to ADDRESS and the other specified properties set to the specified values.

Properties

IPAddress

IP address of the USRP® device

Specify the logical network location of the USRP® device as a string. The default value is '192.168.10.2'. You can find all connected USRP® radios with the findsdru function.

CenterFrequencySource

Source of the center frequency

Specify the source of the center frequency as one of Property | Input port. The default is Property. Set CenterFrequencySource to Input port to specify the center frequency value using an input to the step method. Set CenterFrequencySource to Property to specify the center frequency value using the CenterFrequency property.

CenterFrequency

Desired center frequency in Hertz

Specify the desired center frequency as a double-precision, nonnegative, finite scalar. The default is 2.45 Gigahertz. The valid range of values for this property depends on the RF daughter card of the USRP® device. This property applies when you set the CenterFrequencySource to Property. This property is tunable.

ActualCenterFrequency

Actual center frequency in Hertz.

Report the actual center frequency of the daughterboard. Desired and actual center frequency can be slightly different due to quantization. This property is read-only.

LocalOscillatorOffsetSource

Source of local oscillator offset

Specify the source of the local oscillator offset as one of Property| Input port. The default is Property. When you set the LocalOscillatorOffsetSource property to Input port the object specifies the local oscillator offset value using an input to the step method. When you set LocalOscillatorOffsetSource to Property the object specifies the local oscillator offset value using the LocalOscillatorOffset property.

LocalOscillatorOffset

Local oscillator offset in Hertz

Specify the desired local oscillator offset frequency as a double precision scalar. The default is 0 Hz. The valid range of this property depends on the RF daughterboard of the USRP® device. This property applies when you set the LocalOscillatorOffsetSource to Property. This property is tunable.

ActualLocalOscillatorOffset

Actual local oscillator offset in Hertz

Report the actual local oscillator offset of the daughterboard. Desired and actual offset can be slightly different due to quantization. This property is read-only.

GainSource

Source of gain

Specify the source of the gain as one of Property | Input port. The default is Property. Set GainSource to Input port to specify the gain value using an input to the step method. Set GainSource to Property to specify the gain value using the Gain property.

Gain

Specify the overall gain of the USRP® hardware transmitter data path.

Specify the overall gain, both analog and digital, of the USRP® hardware transmitter data path as a double scalar using decibels. The default is 8. The valid range of this property depends on the RF daughterboard of the USRP® device. This property applies when you set the GainSource to Property. This property is tunable.

ActualGain

Actual gain in decibels

Report the actual gain of the daughterboard. Desired and actual gain can be slightly different due to quantization. This property is read-only.

InterpolationFactorSource

Source of the interpolation factor

Specify the source of the interpolation factor as one of Property | Input port. The default is Property. Set InterpolationSource to Input port to specify the interpolation factor value using an input to the step method. Set InterpolationSource to Property to specify the interpolation factor value using the InterpolationFactor property.

InterpolationFactor

Interpolation factor

Specifies the interpolation factor for the SDRu Transmitter. The default is 512. Select an integer value within the following ranges:

  • 4 to 128

  • 128 to 256 (values in this range must be even)

  • 256 to 512 (values in this range must be evenly divisible by 4)

This property applies when you set the InterpolationFactorSource to Property. This property is tunable.

ActualInterpolationFactor

Actual interpolation factor

Report the actual interpolation factor of the daughterboard. Desired and actual interpolation factors can be slightly different due to quantization. This property is read-only.

UnderrunOutputPort

Enable an output port that indicates dropped packets

Set this property to true so that the step method outputs a value to indicate if one or more packets is dropped during host transmission of USRP® hardware data.

  • Zero indicates no data loss

  • A positive number indicates that underruns occurred

The default value for UnderrunOutputPort is false, which means that the port is not enabled and no information about dropped packets is displayed.

This port is a useful diagnostic tool to determine real time operation of the System objectTM. If your model is not running in real time, you can increase the interpolation factor to approach or achieve real-time performance.

EnableBurstMode

Turn burst mode on or off

When set to true, this property produces a set of contiguous frames without an overrun or underrun to the radio. This setting can help simulate models that cannot run in real time. When enabled, specify the desired amount of contiguous data using the NumFramesInBurst property. The default value is false.

NumFramesInBurst

Number of frames in contiguous burst

This property is valid when EnableBurstMode property is set to true. The default number of frames in a burst is 100.

Examples

Configure a USRP® board with a IP address of 192.168.10.2 to transmit at 2.5 GHz with an interpolation factor of 256.

    hSDRu = comm.SDRuTransmitter(...
              'IPAddress', '192.168.10.2', ...
              'CenterFrequency', 2.5e9, ...
              'InterpolationFactor', 256);
    hMod = comm.DPSKModulator('BitInput',true);
    for counter = 1:20
      data = randi([0 1], 30, 1);
      modSignal = step(hMod, data);
      step(hSDRu, modSignal);
    end

For an additional demonstration that use this System objectTM, see the following example:

  • FRS/GMRS Walkie-Talkie Transmitter with USRP® Hardware: To open this example file, enter edit sdruFRSGMRSTransmitter at the MATLAB command line.

Algorithms

This object implements the algorithm, inputs, and outputs described on the SDRu Transmitter block reference page. The object properties correspond to the block parameters.
